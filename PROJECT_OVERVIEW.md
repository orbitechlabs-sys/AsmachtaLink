# מערכת ניהול הסמכות · חטיבה 228 (AsmachtaLink)

## מה זו המערכת

זוהי מערכת פנימית לניהול **הסמכות וקורסים** ברמת חטיבה. היא מרכזת במקום אחד את כל
מחזור החיים של הסמכה — מרגע שגדוד מבקש קורס, דרך פתיחת ההסמכה, שיבוץ החיילים,
מעקב אחרי מקומות פנויים, ועד סגירה ואישור מי עבר. המטרה היא שהחטיבה תראה תמונה
אחת מסודרת מול כל הגדודים שתחתיה, במקום גיליונות אקסל ווטסאפים מפוזרים.

הממשק כולו בעברית (RTL), והמערכת בנויה סביב שני סוגי משתמשים:

- **חטיבה (brigade)** — הרשאות ניהול מלאות: פותחת הסמכות, מאשרת רישום חיילים,
  מטפלת בדרישות הגדודים, ומפיקה דוחות.
- **גדוד (battalion)** — רואה את ההסמכות הפתוחות, רושם חיילים, ומגיש דרישות לקורסים.
  גדוד רואה בעיקר את הנתונים שלו עצמו.

מעבר בין תפקידים נעשה דרך "מחליף תפקיד" בראש המסך (נשמר ב-cookie), כך שאפשר לבדוק
בקלות איך המערכת נראית מזווית של גדוד מסוים מול זווית החטיבה.

## מושגי הליבה

- **הסמכה (Certification)** — אירוע קורס בודד: שם, מיקום/גורם מוסמך, תאריך התחלה
  וסיום, מספר מקומות, וסטטוס. סטטוסים אפשריים: טיוטה, פתוחה להרשמה, מלאה, סגורה,
  בביצוע, בוצעה, בוטלה. יש מעברי-סטטוס חוקיים בלבד (למשל אי-אפשר לחזור מ"בוצעה").
- **רשומים (Roster)** — החיילים המשובצים להסמכה: שם, מספר אישי, טלפון, גדוד, ומצב
  (נרשם / אושר / השתתף / עבר וכו'). חייל יכול להיות מסומן כ**עתודה** — הוא לא נספר
  במניין המקומות התפוסים.
- **דרישות גדודים (Requests)** — גדוד מבקש קורס מסוג מסוים. החטיבה מטפלת בדרישה,
  ויכולה לפתוח ממנה הסמכה ישירות (הדרישה מקושרת להסמכה שנוצרה).
- **בנק הסמכות (Templates)** — תבניות קבועות לסוגי קורסים חוזרים (מיקום ברירת-מחדל,
  מספר מקומות, דרישות מקדימות, ציוד, אנשי קשר וכו'), כדי לפתוח הסמכה חדשה במהירות.
- **פערים (Gaps)** — מעקב אחרי כמה אנשי-מקצוע חסרים לכל גדוד בכל תחום, וכמה כבר
  נשלחו. כשמאשרים סיום הסמכה, הפער של הגדודים הרלוונטיים יורד אוטומטית לפי מי שעבר.
- **מכסות גדוד (Quotas)** — הקצאת מקומות בהסמכה לגדוד ספציפי.
- **מיסים (Taxes)** — דרישות תפקיד שיש למלא עבור הסמכה מסוימת.
- **התראות (Notifications)** — נוצרות אוטומטית באירועים (הסמכה נפתחה, נסגרה להרשמה,
  בוטלה, חייל נוסף וכו') ומכוונות לתפקיד הרלוונטי.
- **היסטוריית סטטוס (Status history)** — כל שינוי סטטוס נרשם לצורכי מעקב וביקורת.

## מסכים עיקריים

| מסך | מה יש בו |
|-----|----------|
| **לוח שנה** | תצוגת חודש / אג'נדה / גאנט של כל ההסמכות לאורך הזמן, עם סינון |
| **הסמכות** | רשימת כל ההסמכות, יצירה/עריכה, ניהול רשומים, שינוי סטטוס, אישור סיום |
| **דרישות גדודים** | הגשת דרישות, טיפול חטיבתי, ופתיחת הסמכה מתוך דרישה |
| **בנק הסמכות** | ניהול התבניות לקורסים חוזרים |
| **גדודים** | תצוגה לפי גדוד, כולל פערים ומקומות |
| **דוחות** | דוחות מסכמים (לפי חודש, הסמכות שבוצעו, פערים לפי גדוד, ועוד) עם ייצוא ל-Excel/PDF |

## ארכיטקטורה טכנית

- **Framework:** Next.js 16 (App Router) עם React 19 ו-TypeScript.
- **מסד נתונים:** Supabase (PostgreSQL). ההתחברות דרך `pg` עם connection pooler,
  ומוגדרת ב-`.env.local` (`DATABASE_URL` לאפליקציה, `DIRECT_URL` להרצת migrations).
- **גישה לנתונים:** שכבת repositories ב-[lib/db/repositories/](lib/db/repositories/)
  שמריצה SQL ישיר מול Postgres (`query` / `execute` / `withTransaction` ב-
  [lib/db/client.ts](lib/db/client.ts)). אין ORM.
- **API:** Route Handlers תחת [app/api/](app/api/) — REST פנימי לכל ישות.
- **UI:** רכיבי shadcn/ui מעל Tailwind CSS v4, Radix UI, ואייקוני lucide.
- **הרשאות:** לוגיקת התפקידים ב-[lib/auth/permissions.ts](lib/auth/permissions.ts);
  התפקיד הנוכחי נקרא מ-cookie ב-[lib/auth/current-role.ts](lib/auth/current-role.ts).
- **ולידציה:** סכמות Zod ב-[lib/validation/](lib/validation/).

> הערה: קיים גם קובץ SQLite מקומי (`data/certifications.db`) משלב פיתוח מוקדם, אבל
> הסכמה בפועל של האפליקציה היום היא ה-Postgres/Supabase שב-
> [migrations/postgres/001_init.sql](migrations/postgres/001_init.sql).

## סכמת מסד הנתונים (טבלאות עיקריות)

- `battalions` — הגדודים (קוד, שם, צבע, פעיל/לא-פעיל)
- `certifications` — ההסמכות
- `roster_entries` — החיילים הרשומים להסמכות
- `certification_templates` — תבניות בנק ההסמכות
- `battalion_requests` — דרישות הגדודים
- `certification_battalion_quotas` — הקצאת מקומות לגדוד
- `certification_prerequisites` / `certification_taxes` — דרישות מקדימות ומיסים
- `certification_gap_rows` / `certification_gap_values` — טבלת הפערים לפי גדוד
- `notifications` — התראות
- `status_history` — יומן שינויי סטטוס
- `course_colors` — צבע קבוע לכל סוג קורס בלוח השנה

## הרצה מקומית

```bash
npm install
npm run dev        # מריץ את האפליקציה על http://localhost:3000
```

סקריפטים שימושיים (`package.json`):

- `npm run db:migrate` — הרצת ה-schema על Supabase (`migrations/postgres/001_init.sql`)
- `npm run seed` — זריעת הגדודים
- `npm run seed:templates` — זריעת בנק ההסמכות

נדרש קובץ `.env.local` עם `DATABASE_URL` ו-`DIRECT_URL` של Supabase (ראו `.env.example`).

## מצב הנתונים כרגע

הועלו למערכת נתונים אמיתיים מקובץ סיכום חיצוני: **10 הסמכות** ו-**65 חיילים רשומים**
(כולל 3 בעתודה) פרוסים על הגדודים 5030, 6228, 8207 ו-9308. מספרי ה"רשומים/מתוך"
תואמים למקור. שורות "טרם נשלחו שמות" (הקצאות שעדיין ריקות) לא נכנסו כרשומים — המידע
עליהן נשמר בהערות ההסמכה.
